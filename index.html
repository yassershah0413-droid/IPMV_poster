◈ What is Pattern Recognition?

Pattern Recognition is the automated process by which a machine perceives, identifies, and classifies patterns in data — images, signals, or text — by learning from examples. It forms the core of Machine Vision and enables systems to make decisions the way humans do: by recognizing meaningful structure in raw input.

Goal: Assign an input (image, sensor reading) to one of several predefined classes or discover natural groupings in unlabelled data.

⊛ Pattern Recognition Algorithms
01
Statistical / Bayesian Methods
Use probabilistic models to assign class labels. Maximum Likelihood & MAP estimation minimise classification error using prior knowledge.
Probabilistic
02
K-Nearest Neighbour (KNN)
Non-parametric: assigns label based on majority vote among K closest training samples in feature space. Simple, effective for small datasets.
Instance-Based
03
Support Vector Machine (SVM)
Finds optimal hyperplane maximising margin between classes. Kernel trick extends it to non-linear boundaries in high-dimensional spaces.
Discriminative
04
Convolutional Neural Network (CNN)
Learns hierarchical features (edges → shapes → objects) via conv layers. State-of-the-art for image classification, detection, and segmentation tasks.
Deep Learning
05
Template Matching & Structural
Matches input against stored prototypes. Syntactic methods use grammars/graphs to describe spatial relationships between primitives.
Structural
⊡ Object Detection & Recognition
Object detection locates and identifies multiple objects within an image simultaneously — combining localisation (bounding box) and classification (label).
Two-Stage Detectors
R-CNN → Fast R-CNN → Faster R-CNN. Region proposals + classification. Higher accuracy, slower speed.
One-Stage Detectors
YOLO, SSD — direct bounding box prediction. Faster, suitable for real-time applications.

⇅ Learning Paradigms
Supervised
  • Labelled training data
  • Learns decision boundary
  • SVM, KNN, CNN
  • High accuracy if data rich
  • Requires annotation effort
Unsupervised
  • No labels required
  • Discovers hidden clusters
  • K-Means, DBSCAN
  • Scalable to large data
  • Interpretation harder
❖ Key Terminology
Class
Category to which a pattern belongs (e.g., "cat", "dog")
Feature Vector
Numerical representation of an object used for classification
Training Set
Labelled examples used to train the classifier model
Overfitting
Model memorises training data, fails on unseen samples
Decision Boundary
Surface separating classes in the feature space
Generalisation
Model's ability to perform well on new, unseen data
